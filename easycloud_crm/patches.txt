# easycloud_crm/patches.txt
#
# WHAT A "PATCH" IS: a one-time migration script that runs automatically the
# NEXT TIME `bench migrate` runs, then never runs again on that same site
# (Frappe records which patches have already executed in the Patch Log
# doctype). Used for anything a plain doctype-JSON-field-change can't do --
# here, that's things like "create this Kanban Board if it doesn't exist" or
# "grant this permission" -- one-off setup actions, not ongoing behaviour.
# Every patch file below lives in easycloud_crm/patches/ and must define a
# top-level execute() function, which is what actually runs.
#
# IMPORTANT: because each patch only ever runs ONCE per site, editing an
# OLD patch file after it's already run does NOTHING on existing sites --
# only a brand-new site (or one that never ran it yet) would see the change.
# If you need to fix or extend something an old patch already did, write a
# NEW patch instead of editing an old one.
# ==============================================================================
